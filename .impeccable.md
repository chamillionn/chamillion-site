## Design Context

### Users
- **Primary**: Gente con conocimiento base en finanzas, curiosa por cripto/DeFi, escéptica del "criptobroísmo" — buscan divulgación seria, no hype.
- **Secondary**: Traders TradFi jóvenes que están aprendiendo y quieren una fuente de información rigurosa.
- **Hub members**: Suscriptores de pago — lo suficientemente comprometidos para pagar por acceso a herramientas, análisis profundo y la cartera de Chamillion. Llegan buscando claridad y confianza. Contexto: desde checks rápidos en móvil ("¿alguna operación nueva?") hasta sesiones de análisis profundo en escritorio.
- **Context**: Leen desde el móvil y escritorio. Valoran la sustancia sobre la forma. No quieren sentirse dentro de un funnel de ventas.

### Brand Personality
- **3 palabras**: Elegante, Calmado, Autoritativo.
- **Voz**: First-principle thinker. No sigue modas, no usa jerga inflada, no promete rendimientos. Explica con claridad y deja que el lector saque sus conclusiones. Confianza silenciosa — como un inversor experimentado, no como un exchange de cripto.
- **Emotional goals**: Confianza y curiosidad — "esto es serio, quiero seguir leyendo". En el Hub: "confío en esta persona y en lo que me muestra".

### Aesthetic Direction
- **Tone**: Tipografía editorial, densidad de información controlada, sin artificios visuales gratuitos.
- **Dark mode**: Identidad principal — oscuro frío con acento steel blue, serif para jerarquía, monospace para datos.
- **Light mode**: Warm beige/parchment — no clínico blanco. Necesita la misma calidad que el dark.
- **Anti-references**: Plataformas de cursos/membresías de trading (medusacapital.xyz y similares) — todo lo que huele a "comunidad exclusiva de traders", gradientes neón, badges de urgencia, countdowns, testimonios fabricados. Tampoco corporate gray SaaS (Jira/Salesforce). Tampoco minimalismo vacío que parece un wireframe.
- **References**: La UI actual de chamillion.site es la referencia canónica. El Hub es una continuación, no un rediseño. Bloomberg Terminal (densidad de datos con claridad), Financial Times (editorial serio), The Generalist (newsletter limpia y bien tipografiada).

### Design Principles
1. **Credibilidad a través del oficio** — cada detalle (spacing, tipografía, formato de datos) refuerza que una persona seria y competente construyó esto. Alineaciones descuidadas o spacing inconsistente erosionan la confianza.
2. **Densidad con jerarquía** — mostrar datos generosamente pero guiar la mirada. Usar tamaño, peso, opacidad y spacing para crear orden de lectura claro. El usuario nunca debería preguntarse "¿dónde miro?"
3. **Continuidad, no novedad** — el Hub debe sentirse como el mismo producto que la homepage y la newsletter. Mismas fuentes, misma lógica de color, misma cadencia de animación. No experimentos de diseño — extender lo que funciona.
4. **Calidez en la contención** — el steel-blue, el parchment del light mode, los serifs en headings — dan personalidad sin ser ruidosos. Mantener esta calidez. Evitar grises puros y blancos stark.
5. **Cada píxel se gana su sitio** — no hay elementos decorativos que no comuniquen. Si un borde, sombra o animación existe, debe servir a jerarquía, feedback o orientación espacial.
6. **Light mode es ciudadano de primera** — no es un afterthought; debe tener la misma calidad y cuidado que el dark.

### Technical Constraints
- **Styling:** CSS Modules + CSS custom properties. No Tailwind, no CSS-in-JS.
- **Fonts:** Playfair Display (headings), Instrument Serif (brand/accents), Outfit (body/UI), Source Sans 3 (secondary body), JetBrains Mono (badges/labels), DM Mono (code/secondary mono).
- **Colors:** Usar CSS variables existentes (`--steel-blue`, `--bg-card`, `--text-primary`, etc.). Nunca hardcodear colores fuera de `globals.css`.
- **Radius tokens:** `--radius-xs` (4px), `--radius-sm` (6px), `--radius-md` (10px), `--radius-lg` (12px).
- **Transitions:** `--transition-fast` (150ms), `--transition-normal` (250ms).
- **Motion:** Siempre respetar `prefers-reduced-motion`. Usar keyframes existentes (`fadeIn`, `fade-slide-up`, etc.) antes de crear nuevos.
- **Accessibility:** Focus-visible outlines (2px steel-blue), skip-to-content link, semantic HTML, WCAG AA contrast (en light mode steel-blue se deepenea a `#2b5d73`).
- **Patterns to follow:** Admin shell (sidebar + main) para estructura de layout. Monospace uppercase badges para status/labels. `rgba(var(--steel-blue-rgb), opacity)` para backgrounds tintados. Gradient dividers (`linear-gradient(90deg, transparent, var(--border), transparent)`).
